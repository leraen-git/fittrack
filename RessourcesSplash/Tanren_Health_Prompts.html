Strategy

Why pre-prompts matter

Le prompt système iOS/Android est brutal et anonyme. Sans contexte, les utilisateurs refusent à 60-70% par défaut. Le pre-prompt Tanren-branded explique le bénéfice avant que le système ne demande, ce qui transforme le système en formalité plutôt qu'en barrière.

User clicks "Activer"

Sur le pre-prompt Tanren. Il a déjà accepté mentalement. Aucun appel système n'a encore eu lieu.

System prompt fires

Tanren appelle requestAuthorization(). Le prompt iOS apparaît. Le user clique "OK" car il sait déjà pourquoi.

Action executes

La permission est accordée. Tanren effectue l'action prévue (publier la session, lire le poids, etc.) sans transition.

V1 scope

Sleep data reading is explicitly out of scope for V1. Resting heart rate is opt-in only (no proactive prompt — only triggered if the user manually enables it in Settings). The two prompts above are the only proactive permission requests in the V1 launch.
